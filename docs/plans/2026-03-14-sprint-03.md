# Sprint 3: 경제 섹션 5~8 완성 + 부동산 DB 연동 + KB지수 + 권역별 차트 구현 계획

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 탭1 경제 섹션 5~8(무역/고용/통화/인구)을 실제 ECOS 데이터로 완성하고, 탭2 부동산 탭에 PostgreSQL DB 연동을 통해 KB지수 차트와 4개 권역별 매매가/전세가 차트를 렌더링한다.

**Architecture:** 경제 섹션 5~8은 기존 `EconomyIndicatorService.getIndicators()` + `EconomyApiController` 패턴을 그대로 재사용한다. 부동산 탭은 `StatSigunguYymm`, `StatLeaseSigungu` 두 JPA 엔티티와 QueryDSL CustomRepository를 새로 추가하고, `RealEstateService`가 동적 쿼리로 권역/면적타입 필터링을 담당한다. KB지수는 기존 ECOS API 경로(`EcosApiService`)를 그대로 활용한다. 프론트엔드는 기존 `common.js`의 `createLineChart` / `createBarChart` 함수를 재사용하고 부동산 전용 AJAX 로직을 `realestate.js`로 분리한다.

**Tech Stack:** Spring Boot 3.3.2, Spring Data JPA + QueryDSL 5.0.0 (Jakarta), PostgreSQL, Jackson, Chart.js 4.4.3, Bootstrap 5.3.3, JUnit 5 + Mockito + MockMvc

경상수지는 양/음 값이 모두 존재하므로 막대 차트(기존 `createBarChart`)를 사용한다. 수출/수입은 금액(백만달러 단위)이 경상수지와 단위가 달라 별도 이중 Y축 또는 별도 차트로 분리한다. 이번 구현에서는 단순화를 위해 **경상수지 막대 차트 1개 + 수출/수입 라인 차트 1개**를 같은 카드 안에 나란히 렌더링한다.

```javascript
function loadTradeChart() {
    fetch('/krEconoMon/api/economy/trade')
        .then(res => {
            if (!res.ok) throw new Error('무역 데이터 로드 실패: ' + res.status);
            return res.json();
        })
        .then(data => {
            // 경상수지(index 0): 막대 차트, 수출/수입(index 1,2): 라인+이중 Y축
            const currentAccountDataset = { ...data.datasets[0], type: 'bar' };
            const tradeData = {
                labels: data.labels,
                datasets: [
                    currentAccountDataset,
                    data.datasets[1],
                    data.datasets[2]
                ]
            };
            const options = {
                scales: {
                    y: {
                        type: 'linear', display: true, position: 'left',
                        title: { display: true, text: '경상수지 (백만달러)' }
                    },
                    y1: {
                        type: 'linear', display: true, position: 'right',
                        title: { display: true, text: '수출/수입 (백만달러)' },
                        grid: { drawOnChartArea: false }
                    }
                }
            };
            createLineChart('tradeChart', tradeData, options);
        })
        .catch(err => console.error('무역 데이터 로드 실패:', err));
}
```

`DOMContentLoaded` 핸들러의 `shown.bs.tab` 이벤트 블록과 active 탭 블록 양쪽에 `loadTradeChart();` 를 추가한다.

### Step 4: QueryDSL CustomRepository 구현

> Q클래스는 Step 6 `compileJava` 실행 후 사용 가능해진다. 먼저 인터페이스/Repository만 작성하고, Impl은 compileJava 후 작성하거나 컴파일 에러를 무시하고 작성 가능하다.

`src/main/java/com/example/kreconomonmon/repository/StatSigunguYymmRepositoryImpl.java`:

```java
package com.example.kreconomonmon.repository;

import com.example.kreconomonmon.entity.QStatSigunguYymm;
import com.example.kreconomonmon.entity.StatSigunguYymm;
import com.querydsl.jpa.impl.JPAQueryFactory;
import lombok.RequiredArgsConstructor;

import java.time.LocalDate;
import java.util.List;

@RequiredArgsConstructor
public class StatSigunguYymmRepositoryImpl implements StatSigunguYymmRepositoryCustom {

    private final JPAQueryFactory queryFactory;

    @Override
    public List<StatSigunguYymm> findByCodesAndAreaType(List<String> sigunguCodes, String useAreaType) {
        QStatSigunguYymm q = QStatSigunguYymm.statSigunguYymm;

        String tenYearsAgo = String.valueOf(LocalDate.now().getYear() - 10) + "01";

        return queryFactory
                .selectFrom(q)
                .where(
                    q.sigunguCode.in(sigunguCodes),
                    q.useAreaType.eq(useAreaType),
                    q.dealYymm.goe(tenYearsAgo)
                )
                .orderBy(q.sigunguCode.asc(), q.dealYymm.asc())
                .fetch();
    }
}
```

```java
    private static final List<String> COLORS = List.of("#007bff", "#28a745", "#fd7e14");

    /**
     * 권역별 매매가 차트 데이터 반환.
     * sigunguCodes 순서대로 데이터셋이 생성된다.
     */
    public ChartDataResponse getPriceChartData(List<String> sigunguCodes, String useAreaType) {
        List<StatSigunguYymm> rows =
            tradeRepository.findByCodesAndAreaType(sigunguCodes, useAreaType);

        return buildChartResponse(
            sigunguCodes,
            rows.stream().collect(Collectors.groupingBy(StatSigunguYymm::getSigunguCode)),
            rows.stream()
               .filter(r -> r.getSigunguCode().equals(sigunguCodes.get(0)))
               .map(StatSigunguYymm::getDealYymm)
               .collect(Collectors.toList()),
            sigunguCodes
        );
    }

    private ChartDataResponse buildChartResponse(
            List<String> sigunguCodes,
            Map<String, ?> dataByCode,
            List<String> labels,
            List<String> codes) {

        List<ChartDataResponse.Dataset> datasets = new ArrayList<>();

        for (int i = 0; i < codes.size(); i++) {
            String code = codes.get(i);
            String color = COLORS.get(i % COLORS.size());
            Object groupData = dataByCode.get(code);

            List<Double> values;
            if (groupData instanceof List<?> list) {
                values = list.stream()
                    .map(item -> {
                        if (item instanceof StatSigunguYymm t)
                            return t.getAvgPrice() != null ? t.getAvgPrice().doubleValue() : null;
                        if (item instanceof StatLeaseSigungu l)
                            return l.getAvgDeposit() != null ? l.getAvgDeposit().doubleValue() : null;
                        return null;
                    })
                    .collect(Collectors.toList());
            } else {
                values = new ArrayList<>(Collections.nCopies(labels.size(), null));
            }

            datasets.add(ChartDataResponse.Dataset.builder()
                .label(getDistrictName(code))
                .data(values)
                .borderColor(color)
                .backgroundColor(color)
                .fill(false)
                .build());
        }

        return ChartDataResponse.builder()
            .labels(labels)
            .datasets(datasets)
            .build();
    }

    /**
     * 권역별 전세가 차트 데이터 반환.
     */
    public ChartDataResponse getLeaseChartData(List<String> sigunguCodes, String useAreaType) {
        List<StatLeaseSigungu> rows =
            leaseRepository.findByCodesAndAreaType(sigunguCodes, useAreaType);

        List<String> labels = rows.stream()
            .filter(r -> r.getSigunguCode().equals(sigunguCodes.get(0)))
            .map(StatLeaseSigungu::getDealYymm)
            .collect(Collectors.toList());

        Map<String, List<StatLeaseSigungu>> grouped =
            rows.stream().collect(Collectors.groupingBy(StatLeaseSigungu::getSigunguCode));

        List<ChartDataResponse.Dataset> datasets = new ArrayList<>();
        for (int i = 0; i < sigunguCodes.size(); i++) {
            String code = sigunguCodes.get(i);
            String color = COLORS.get(i % COLORS.size());
            List<StatLeaseSigungu> group = grouped.getOrDefault(code, List.of());

            List<Double> values = group.stream()
                .map(l -> l.getAvgDeposit() != null ? l.getAvgDeposit().doubleValue() : null)
                .collect(Collectors.toList());

            datasets.add(ChartDataResponse.Dataset.builder()
                .label(getDistrictName(code))
                .data(values)
                .borderColor(color)
                .backgroundColor(color)
                .fill(false)
                .build());
        }

        return ChartDataResponse.builder()
            .labels(labels)
            .datasets(datasets)
            .build();
    }
}
```

## 주의사항 및 트러블슈팅

### DB 테이블 컬럼명 불일치
실제 PostgreSQL 테이블 컬럼명이 엔티티에 정의된 것과 다를 수 있다. Task 5 착수 전 반드시 DB 스키마를 확인한다:

```sql
\d stat_sigungu_yymm
\d stat_lease_sigungu
```

컬럼명이 다르면 엔티티의 `@Column(name = "실제컬럼명")`을 수정한다.

### QueryDSL Q클래스 생성 실패
Q클래스가 생성되지 않으면:
```bash
./gradlew clean compileJava
```
`build.gradle`의 `annotationProcessor` 설정을 확인한다.

### ECOS API KB지수 응답 없음
KB지수 ECOS 코드(`901Y062`, `901Y063`)가 실제 API에서 지원되지 않으면:
- ECOS Open API 포털에서 해당 통계코드 검색 확인
- 응답이 비어 있어도 빈 차트로 렌더링되므로 오류는 발생하지 않음
