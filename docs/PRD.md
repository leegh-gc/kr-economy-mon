### 2-1. 외부 데이터 소스

| 소스 | 용도 | 비고 |
|------|------|------|
| 한국은행 ECOS API | 경제 지표 데이터 | `http://ecos.bok.or.kr/api/StatisticSearch` |
| PostgreSQL DB | 서울 부동산 거래 데이터 및 경제지표 데이터 저장 | `jdbc:postgresql://leeghhome.iptime.org:5432/aptprice_data` |
| Gemini API | 경제/부동산 현황 분석 + 컷툰 생성 | `gemini-3-flash` 모델 사용 (컷툰) |

### 2-2. 캐싱 & 재분석 정책 (공통)

- 분석 결과(텍스트) 및 컷툰 이미지를 **로컬 스토리지 또는 서버 캐시에 저장**
- 페이지 로드 시:
  1. 기존 캐시 데이터와 **신규 데이터 여부 비교**
  2. **신규 데이터 없음** → 기존 분석 결과 & 컷툰 그대로 표시
  3. **신규 데이터 있음** → Gemini API 재호출 → 분석 & 컷툰 재생성 후 캐시 갱신
- Gemini API는 **불필요한 중복 호출 금지**

### 4-6. 권역별 표시 구성

각 권역(탭 or 섹션)을 클릭하면 해당 3개 구 데이터 표시:

#### A. 매매가/전세가 월별 통계 차트 (10년치)

```sql
-- 매매가
SELECT sigungu_name, deal_yymm, get_code_name(use_area_type),
       avg_price, total_count
FROM stat_sigungu_yymm
WHERE sigungu_code = :code AND use_area_type = 'UA04'
  AND deal_year BETWEEN '2017' AND '2026'
ORDER BY deal_yymm DESC;

-- 전세가
SELECT sigungu_name, deal_yymm, get_code_name(use_area_type),
       avg_deposit, total_count
FROM stat_lease_sigungu
WHERE rent_gbn = '0' AND sigungu_code = :code AND use_area_type = 'UA04'
  AND deal_year BETWEEN '2017' AND '2026'
ORDER BY deal_yymm DESC;
```

- 차트 유형: **라인 차트** (avg_price / avg_deposit 기준)
- 3개 구를 **한 차트**에 멀티 라인으로 표현 or 구별 차트 3개 나란히

#### B. 올해 평균 매매가/전세가 TOP 5 아파트 목록

```sql
-- 매매 TOP 5
SELECT deal_year, sigungu_name || ' ' || land_dong AS 위치,
       apt_name, build_year, get_code_name(use_area_type),
       min_price, avg_price, max_price, trade_count
FROM rank_uatype_sigungu
WHERE deal_year = '2026' AND rank_type = 0
  AND sigungu_code = :code AND use_area_type = 'UA04'
ORDER BY avg_price DESC LIMIT 5;

-- 전세 TOP 5
SELECT deal_year, sigungu_name || ' ' || land_dong AS 위치,
       apt_name, build_year, get_code_name(use_area_type),
       min_deposit, avg_deposit, max_deposit, trade_count
FROM rank_uatype_sigungu_lease
WHERE deal_year = '2026' AND rank_type = 0 AND rent_gbn = '0'
  AND sigungu_code = :code AND use_area_type = 'UA04'
ORDER BY avg_deposit DESC LIMIT 5;
```

- 표시 형식: **테이블 리스트** (아파트명, 위치, 건축연도, (최고가|평균가|최저가)--세로, 거래건수)
- 매매 TOP5 + 전세 TOP5 구분 표시

## 5. 미확정 사항 (착수 전 확인 필요)

| # | 항목 | 확인 필요 내용 |
|---|------|--------------|
| 1 | ECOS API 인증키 | API Key 발급 여부 및 환경변수 관리 방식 |
| 2 | Gemini API 키 | 키 발급 여부, 사용할 모델명 정확히 확인 (nano 버전 이미지 생성 지원 여부) |
| 3 | DB 접속 방식 | 서버 사이드(Node.js/Python 백엔드) 또는 프론트엔드 직접 연결 여부 |
| 4 | `use_area_type` | 기본 표시 면적 타입 (UA04 고정 or 사용자 선택 가능) |
| 5 | 캐시 저장 위치 | 브라우저 localStorage / 서버 DB / Redis 등 |
| 6 | 배포 환경 | SPA(React/Vue) 단독 or Next.js/FastAPI 등 풀스택 |
| 7 | 환율 차트 단위 | JPY는 100엔 단위 → 이중 Y축 or 정규화 처리 방식 |
| 8 | 컷툰 형식 | 몇 컷 구성인지 (2컷, 4컷 등), 스타일 가이드 |

## 6. 권장 개발 순서

```
1단계: 프로젝트 초기 세팅 (탭 레이아웃, 라우팅)
2단계: ECOS API 연동 → 탭1 차트 렌더링
3단계: PostgreSQL 연동 → 탭2 KB지수 차트 + 권역별 통계
4단계: TOP5 아파트 테이블 구현
5단계: 캐시 레이어 구현
6단계: Gemini API 연동 → 분석 텍스트 생성
7단계: Gemini nano 컷툰 생성
8단계: 캐싱 정책(재분석 트리거) 연결 + 전체 QA
```
